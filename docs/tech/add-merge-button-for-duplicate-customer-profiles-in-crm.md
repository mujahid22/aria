# Add Merge Button for Duplicate Customer Profiles in CRM
## Overview
This Business Requirements Document (BRD) outlines the need to introduce a merge button within the CRM interface to address the issue of duplicate customer profiles. This feature will enable users to consolidate duplicate profiles into a single, accurate record, improving data integrity and user efficiency.
## Description
The CRM system currently lacks a feature to merge duplicate customer profiles, leading to data redundancy and inefficiencies in customer management. This requirement introduces a merge button that will allow users to select and combine duplicate profiles into a primary record. The feature will ensure all relevant data, such as contact details and interaction history, is preserved and consolidated.
## Acceptance Criteria
- The merge button is visible and accessible on the duplicate customer profile detection screen.
- Users can select two or more duplicate profiles to merge.
- The system prompts for confirmation before merging the selected profiles.
- Merging combines all relevant data (e.g., contact details, interaction history) into a primary profile.
- The system logs the merge action in the audit trail for tracking.
- Users receive a success notification post-merge, confirming the action.

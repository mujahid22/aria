# Add Quick-Reorder Button to Past Orders
## Overview
This Business Requirements Document (BRD) outlines the need to implement a quick-reorder button for users on the past orders page. This feature aims to enhance user convenience by allowing them to reorder items from their purchase history without manually adding each item to the cart.
## Description
The quick-reorder button will be added to the past orders page, enabling users to reorder items from any completed order with a single click. This feature will streamline the reordering process, improve user experience, and potentially increase sales by reducing friction for repeat purchases.
## Acceptance Criteria
- The quick-reorder button must appear on the past orders page for each completed order.
- Clicking the quick-reorder button must add all items from the selected past order to the user's cart.
- The button must only include items that are currently in stock.
- Users must be notified if any items from the past order are out of stock and cannot be reordered.
- The cart must update to reflect the added items without requiring a page refresh.
- The feature must work on both desktop and mobile views.
